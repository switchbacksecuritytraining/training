Abstract

Windows security rests on the isolation of the kernel and the enforcement of boundaries between rings. For researchers and advanced defenders, knowing where those boundaries break down is the whole job.

This four-day course is an intensive, lab-driven tour of kernel internals, memory management, and privilege elevation. It is grounded in real case studies — you will dissect arbitrary pointer writes and pool corruptions that shipped in production, and work out why each was exploitable rather than merely crashing.

The goal is engineering-level understanding of both the offensive mechanics and the defensive countermeasures they force. You leave able to reason about a new bug class you have never seen before.

Who it's for

  • Vulnerability researchers moving from user-mode analysis into kernel internals.
  • Detection engineers and blue teamers who need exploit primitives to build behavioural monitoring that doesn't false-positive.
  • Reverse engineers working on driver communication, undocumented interfaces, and pool grooming.

What you'll be able to do

  • Audit driver entry points and IOCTL handlers for logic and memory flaws.
  • Navigate kernel memory structures in WinDbg and Ghidra.
  • Turn an arbitrary write primitive into a reliable privilege elevation.
  • Judge whether a given bug class is worth the exploitation effort.
  • Explain what SMEP, HVCI and kCFG actually cost an attacker.

The labs culminate in two complete privilege elevation chains built from two different real-world vulnerabilities.

Outline

Module 1 — Tooling and kernel architecture

  • Kernel debugging environments: WinDbg, VMs, symbol paths that work
    • Reversing the Windows kernel without drowning
    • Where AI-assisted structure discovery helps, and where it lies
  • Subsystems and drivers: object and memory management, dispatch routines, IRPs

Module 2 — Driver attack surface

  • Auditing IOCTL handlers; tracing data from user mode to kernel mode
  • Vulnerability classes: memory corruption, races, use-after-free

Module 3 — Structures and info leaks

  • Files, named pipes, IO rings, and what happens when you corrupt them
  • Known ASLR defeats and the mitigations aimed at them
  • Case study: an arbitrary write primitive, start to finish

Module 4 — Reliability and impact

  • Deterministic versus probabilistic exploitation; estimating effort honestly
  • Achieving kernel read/write: object modification versus code execution

Module 5 — Mitigations

  • SMEP, SMAP, HVCI, kCFG — and their effect on exploitability

Module 6 — Heap internals and advanced corruption

  • Segment heap, lookaside lists, and shaping memory layout deliberately
  • Case study: semi-controlled overwrite to full privilege elevation

Prerequisites

This is not an introduction. You should arrive with:

  • Working experience in C or C++
  • Comfort with a disassembler (Ghidra, IDA) and a debugger
  • Basic knowledge of Windows internals
  • Familiarity with common vulnerability classes

What to bring

A Windows 11 laptop you have administrator rights on, with:

  • 80 GB free disk
  • 16 GB RAM minimum, 32 GB if you have it
  • Hyper-V enabled, or VMware
  • Ghidra, or an equivalent you already know
  • WinDbg — the classic build, installed via Debugging Tools for Windows
  • Optional: an active subscription to a commercial LLM
