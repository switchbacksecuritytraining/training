Abstract

Most security teams inherited Kubernetes rather than chose it, and the gap between running a cluster and defending one is wide. This course closes it.

The work is organised by layer — supply chain, infrastructure, runtime — because that is how the attacks arrive. You will look at what an attacker touches at each one, what evidence they leave, and which controls actually hold versus which only look like they do. AI/ML workloads get their own treatment: they widen the attack surface in ways that most existing cluster hardening never anticipated.

Hands-on throughout, built from real-world scenarios rather than toy clusters.

Who it's for

  • Blue teamers defending clusters they did not design.
  • Cloud security engineers responsible for cluster posture.
  • Security architects reviewing cloud-native designs.
  • DevSecOps practitioners owning the pipeline into production.

What you'll be able to do

  • Map the attack surface of a cluster across supply chain, infrastructure and runtime.
  • Identify the evidence an attacker leaves at each layer.
  • Assess the security posture of AI/ML workloads running on Kubernetes.
  • Distinguish controls that hold from controls that only appear to.
  • Run an architecture review of a cloud-native environment.
