Abstract

When an intrusion happens in someone else's datacentre, the evidence is whatever the platform decided to log. Cloud incident response is therefore a different discipline from the on-premises version — you cannot image the disk, and the artifacts you need may expire before you ask for them.

This course covers detecting, scoping and investigating attacks across Azure and Microsoft 365. The first half works through Azure attacks phase by phase, focusing on the evidence each one leaves. The second half moves to Microsoft 365 — acquiring data by several methods, then parsing, enriching and analysing the Unified Audit Log.

Everything is applied in hands-on labs in a CTF-style environment, including two complete attack scenarios you work end to end.

Who it's for

  • Incident responders whose scope now includes the Microsoft cloud.
  • SOC analysts investigating Azure and M365 alerts.
  • Forensic examiners adapting on-premises tradecraft to cloud evidence.
  • Cloud security engineers who need to know what their logs will and won't tell them.

What you'll be able to do

  • Scope an incident across an Azure and Microsoft 365 estate.
  • Identify Azure attack phases from the evidence each leaves behind.
  • Acquire Microsoft 365 data using multiple methods and tools.
  • Parse, enrich and analyse the Unified Audit Log.
  • Stand up a cloud incident response capability.
