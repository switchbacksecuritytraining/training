Hacking Cryptography

Cryptographic primitives rarely break. The code around them breaks constantly. Two days on how applied cryptography fails in the field — and how to find it before someone else does.

Abstract

Crypto-related bugs remain common across the software industry, and they persist for reasons that have little to do with the mathematics.

Developer misuse

Roughly 83% of cryptographic software vulnerabilities come from application developers misusing secure libraries — not from flaws in the libraries themselves.

Business logic errors

In the smart contract space, business logic errors account for a disproportionately high share of bugs.

Persistent value lost

Despite years of industry advancement, bad actors keep exploiting these code vulnerabilities successfully.

What you'll
be able to do

Find the flaw, then write the exploit

You'll leave able to find common crypto vulnerabilities in software and write crypto exploits for IoT devices and software.

Using case studies from our own pentesting and red teaming engagements, we introduce the core concepts of applied cryptography and show how each one fails in practice.
